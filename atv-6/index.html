TCP/IP - UMA BREVE EXPLICAÇÃO

Esta página tem um simples objetivo em apresentar alguns pontos interessantes sobre o conjunto de protocolos TCP/IP. Texto baseado na página TCP/IP publicado pela Wikipedia.

O TCP/IP é um conjunto de protocolos de comunicação entre computadores em rede. Ele é composto pelo TCP e pelo IP, que são responsáveis por controlar a transmissão de dados e pela interconexão dos dispositivos, respectivamente. O TCP/IP segue o modelo de camadas, onde cada camada tem funções específicas e fornece serviços definidos para as camadas superiores. As camadas mais altas lidam com dados mais abstratos e dependem das camadas mais baixas para tarefas de menor nível de abstração.

HISTÓRIA DO TCP/IP

O TCP/IP foi desenvolvido em 1969 como parte do projeto ARPANET, uma rede experimental criada pelo Departamento de Defesa dos Estados Unidos. O objetivo era permitir a comunicação entre sistemas de computadores dispersos, incluindo organizações militares. O TCP (Transmission Control Protocol) foi projetado para controlar a transmissão de dados, garantindo que eles chegassem corretamente ao destino. Já o IP (Internet Protocol) tinha a função de interconectar os dispositivos, roteando os pacotes de dados pela rede.

Ao longo dos anos, o ARPANET se expandiu e se transformou na Internet que conhecemos hoje. Em 1983, ficou definido que todos os computadores conectados à ARPANET deveriam utilizar o TCP/IP como protocolo de comunicação. Essa decisão foi tomada para facilitar a integração de tecnologias emergentes, como rádio e comunicação por satélite, que não eram suportadas pelo modelo anteriormente utilizado pela ARPANET.

No final dos anos 1980, a Fundação Nacional de Ciências dos EUA começou a construir o NSFNET, um backbone de alta velocidade para interconectar diferentes comunidades de pesquisa e os computadores da ARPANET. Em 1990, o TCP/IP se tornou o protocolo padrão do NSFNET, consolidando sua posição como o protocolo principal da Internet.

Desde então, o TCP/IP tem sido amplamente adotado em todo o mundo e é a base para a comunicação na Internet. Ele permite que computadores e dispositivos em redes diferentes se comuniquem de maneira eficiente, garantindo a entrega confiável dos dados. O TCP/IP continua sendo um dos pilares fundamentais da infraestrutura da Internet e desempenha um papel crucial na conectividade global.

BENEFÍCIOS DO PROTOCOLO TCP/IP

O TCP/IP sempre foi considerado um modelo bastante pesado quando comparado com modelo UDP/IP, uma vez que o TCP/IP tem muito de seu foco voltado para a confiabilidade ao invés de só velocidade, diferente do UDP/IP, que é mais leve e veloz, mas não garante que a informação chegará ao respectivo destino.

Com o desenvolvimento das interfaces gráficas, com a evolução dos processadores e com o esforço dos desenvolvedores de sistemas operacionais em oferecer o TCP/IP para as suas plataformas com performance igual ou às vezes superior aos outros protocolos, o TCP/IP se tornou um protocolo indispensável.

O TCP/IP oferece alguns benefícios, dentre eles:

  • Padronização: um padrão, um protocolo roteável que é o mais completo e aceito protocolo disponível atualmente. Todos os sistemas operacionais modernos oferecem suporte para o TCP/IP e a maioria das grandes redes se baseia em TCP/IP para a maior parte de seu tráfego;
  • Interconectividade: uma tecnologia para conectar sistemas não similares. Muitos utilitários padrões de conectividade estão disponíveis para acessar e transferir dados entre esses sistemas não similares, incluindo FTP (File Transfer Protocol) e Telnet (Terminal Emulation Protocol);
  • Roteamento: permite e habilita as tecnologias mais antigas e as novas a se conectarem à Internet. Trabalha com protocolos de linha como P2P (Point to Point Protocol) permitindo conexão remota a partir de linha discada ou dedicada. Trabalha como os mecanismos IPCs e interfaces mais utilizados pelos sistemas operacionais, como sockets do Windows e NetBIOS;
  • Protocolo robusto: escalável, multiplataforma, com estrutura para ser utilizada em sistemas operacionais cliente/servidor, permitindo a utilização de aplicações desse porte entre dois pontos distantes;
  • Internet: é através da suíte de protocolos TCP/IP que obtemos acesso a Internet. As redes locais distribuem servidores de acesso a Internet (proxy servers) e os hosts locais se conectam a estes servidores para obter o acesso a Internet. Este acesso só pode ser conseguido se os computadores estiverem configurados para utilizar TCP/IP.

CAMADAS DA PILHA DOS PROTOCOLOS INTERNET

O modelo ou arquitetura TCP/IP de encapsulamento busca fornecer abstração aos protocolos e serviços para diferentes camadas de uma pilha de estruturas de dados (ou simplesmente pilha).

CAMADA EXEMPLO
Aplicação HTTP, HTTPS, FTP, DNS Essa parte contém todos os protocolos para um serviço específico de comunicação de dados em um nível de processo-a-processo (por exemplo: como um web browser deve se comunicar com um servidor da web).
Transporte TCP, UDP, SCTP Essa parte controla a comunicação host-a-host. [protocolos como OSPF, que é executado sobre IP, podem também ser considerados parte da camada de rede]
Internet Para TCP/IP o protocolo é IP, MPLS Essa parte é responsável pelas conexões entre as redes locais, estabelecendo assim a interconexão. [protocolos requeridos como ICMP e IGMP é executado sobre IP, mas podem ainda ser considerados parte da camada de rede; ARP não roda sobre IP]
Enlace (Interface com Rede) Essa é a parte responsável por enviar o datagrama recebido pela camada de "Internet" em forma de um quadro através da rede. Tecnologias usadas para as conexões: Ethernet rede com fio e Wi-Fi rede sem fio. No modelo OSI, essa camada também é física, porém, é dividido em duas partes: física e enlace de dados. A física é a parte do hardware (por exemplo os cabos das redes com fio) e a enlace de dados é a parte lógica do hardware: endereço MAC de origem e destino; controle de enlace lógico; controle de acesso ao meio.
Físico A camada de interface de rede ou física é a primeira camada. Também chamada camada de abstração de hardware, tem como função principal a interface do modelo TCP/IP com os diversos tipos de redes (X.25, ATM, FDDI, Ethernet, Token Ring, Frame Relay, sistema de conexão ponto-a-ponto SLIP etc.)

IMPLEMENTAÇÕES

Hoje em dia, a maioria dos sistemas operacionais comerciais já possuem a pilha TCP/IP instalada por padrão, tornando desnecessária a busca por implementações adicionais. Essa pilha está presente em todas as versões do Unix, Linux, Mac OS e Microsoft Windows. A implementação do TCP/IP não requer hardware ou software específicos, pois já existem muitas opções disponíveis.

A maioria dos sistemas operacionais, incluindo os de consumo, já incluem a implementação do TCP/IP. Uma implementação típica inclui os protocolos IP, ARP, ICMP, UDP e TCP. Em alguns casos, também podem ser incluídos o IGMP e o IPv6, além de recursos adicionais de segurança como o IPSec. Outros protocolos, como DNS e DHCP, podem ser facilmente adicionados posteriormente.

Normalmente, os programadores de aplicativos se preocupam apenas com a interface de aplicação e, às vezes, com a camada de transporte. As camadas inferiores são fornecidas pelo conjunto TCP/IP no sistema operacional. A maioria das implementações de IP são acessíveis aos programadores por meio de sockets e APIs.

Existem também implementações específicas, como o Lightweight TCP/IP, projetado para sistemas embarcados, e o KA9Q NOS, um conjunto de protocolos para sistemas de radioamador e computadores pessoais.

No caso de adaptadores de rede, o firmware do microcontrolador lida com as questões de link e é suportado pelo driver de software no sistema operacional. Os componentes físicos abaixo da camada de enlace são normalmente tratados por eletrônicos não programáveis, usando chips ASIC (Application-Specific Integrated Circuit) para cada interface de rede. Roteadores de alto desempenho são baseados principalmente em eletrônicos digitais rápidos não programáveis.
